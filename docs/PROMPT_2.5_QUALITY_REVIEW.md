# Prompt 2.5: Quality Review — оценка описаний и атрибутов

## Задача

После структурной валидации (Prompt 2) нужно **оценить качество контента** — описания, специальности, квалификации, языки. Это субъективная проверка — ты (Claude Code) читаешь данные и выносишь оценку.

---

## Шаг 1: Запусти скрипт-помощник

```bash
python scripts/print_quality.py
```

Он выведет все practitioners и выборку services в удобном формате. Прочитай вывод и оцени по критериям ниже.

Если скрипт недоступен — прочитай файлы напрямую:
```
data/output/practitioners.json
data/output/services.json
```

---

## Шаг 2: Оценка practitioners (ВСЕ)

Для **каждого** practitioner из `practitioners.json` проверь:

### description_i18n.en
- Не пустое? (допустимо пустое если `source: "services_sheet"` — у них нет описаний)
- Осмысленный текст на английском? Не набор символов?
- Нет HTML-тегов (`<br>`, `<p>`, `&nbsp;`)?
- Не обрезано на полуслове?
- Не содержит мусор (lorem ipsum, placeholder text)?

### speciality
- Заполнено? Конкретная специальность (не просто "Doctor")?
- Соответствует описанию?

### languages
- Список реальных языков?
- Нет слипшихся строк ("ENGLISHRUSSIAN" вместо ["English", "Russian"])?
- Нет дубликатов?

### qualifications (primary, secondary, additional)
- Осмысленные? Не дублируют друг друга?
- Не пустые строки внутри массива?

### years_of_experience
- Число от 1 до 50? (null допустим)
- Правдоподобно для данного врача?

### Оценка каждого practitioner
- **OK** — всё в порядке
- **WARN** — мелкие проблемы (пустое описание при source=services_sheet, нет опыта)
- **FAIL** — критичные проблемы (слипшиеся языки, HTML в описании, обрезанный текст)

---

## Шаг 3: Оценка services (выборка ~30 из разных категорий)

Выбери ~30 сервисов из **разных категорий**. Для каждого проверь:

### name_i18n.en
- Нет опечаток? Нет обрезанных слов?
- Имя осмысленное?

### description_i18n.en
- Если заполнено — соответствует названию сервиса? Осмысленное?
- Если пустое — ожидаемо (в CSV нет колонки описаний для большинства сервисов)

### price_note_i18n.en
- Если заполнено — формат корректен?
- "Price excludes VAT" присутствует где нужно?

### price_min / duration_minutes
- Правдоподобные значения? Цена не отрицательная? Duration не 0?

## Шаг 4: Выведи отчёт

```
═══════════════════════════════════════════════
  QUALITY REVIEW REPORT
═══════════════════════════════════════════════

PRACTITIONERS (N total):
  OK:   X
  WARN: Y
  FAIL: Z

  Детали по WARN/FAIL:
  WARN Имя: причина
  FAIL Имя: причина
  ...

SERVICES (M sampled):
  OK:   X
  WARN: Y
  FAIL: Z

  Детали по WARN/FAIL:
  WARN Имя: причина
  ...

DECISION: PASS / NEEDS FIXES
═══════════════════════════════════════════════
```

---

## Правила принятия решения

- **PASS** — 0 FAIL, допустимы WARN
- **NEEDS FIXES** — хотя бы 1 FAIL

Если NEEDS FIXES:
1. Опиши что именно нужно исправить
2. Исправь данные в JSON
3. Повтори проверку

---

## Допустимые WARN (НЕ ошибки)

| Ситуация | Оценка | Причина |
|----------|--------|---------|
| Пустое description при `source: "services_sheet"` | WARN | У этих practitioners нет описаний в источнике |
| Пустое description у services | WARN | В CSV нет колонки описаний |
| `years_of_experience: null` | WARN | Не указано в источнике |
| `additional_qualifications: null` | OK | Необязательное поле |
